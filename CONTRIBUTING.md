# Contributing & getting involved

Thanks for your interest in Concordance. The proficiency-encoding engine lives in a
private repository, so this repo isn't where the application code is built - but there are
still real, useful ways to help, and they genuinely shape the project.

## Ways to help

- **Open an issue.** Bugs in the live app, feature ideas, or questions about the pedagogy
  are all welcome - pick a template from
  [New issue](https://github.com/monsieur-trenton/concordance/issues/new/choose).
- **Try it and give feedback.** Use the app at
  [concordancelearn.com](https://concordancelearn.com/) and tell me what works and what
  doesn't. Teacher feedback in particular changes priorities.
- **Vote on features in-app.** Concordance has a feature-voting board built in - signed-in
  users help decide what gets built next.
- **Share it** with a French teacher, department, or district who'd find it useful.
- **[Sponsor the work](https://github.com/sponsors/monsieur-trenton).** Sponsorship covers
  hosting and API costs and buys the time to build the roadmap properly. See
  [ROADMAP.md](ROADMAP.md) for what's funded next.

## Docs pull requests

Small fixes to the public docs in this repo - typos, broken links, clarity - are welcome
as pull requests. For anything larger, open an issue first so we can talk it through before
you spend time on it.

## A note on conduct

Participation here is covered by the [Code of Conduct](CODE_OF_CONDUCT.md). Be kind; many
people in this community are educators and students.
